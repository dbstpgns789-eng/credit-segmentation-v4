# 프로젝트 발전 과정 상세 문서

## 📌 개요
이 디렉토리는 원천 데이터에서 최종 v4 모델에 이르기까지의 **전체 발전 과정**, **의사결정 근거**, **피처 엔지니어링 전략**을 체계적으로 정리한 문서입니다.

## 🎯 목적

### 1. **투명성 (Transparency)**
- 모든 의사결정 과정을 투명하게 공개
- 왜 특정 피처를 선택했는지, 왜 특정 모델을 사용했는지 명확히 설명

### 2. **재현성 (Reproducibility)**
- 누구나 같은 과정을 따라 동일한 결과를 얻을 수 있도록
- 각 버전의 코드, 데이터, 하이퍼파라미터 기록

### 3. **학습 가치 (Educational Value)**
- 초보자도 프로젝트의 발전 과정을 이해할 수 있도록
- 실패한 시도와 성공한 전략 모두 기록

### 4. **확장성 (Scalability)**
- 향후 v5, v6 개발 시 참고할 수 있는 baseline
- 다른 유사 프로젝트에 적용 가능한 방법론 제시

## 📊 프로젝트 발전 타임라인 요약

| 버전 | 핵심 변화 | Macro F1 | 주요 개선 사항 |
|------|----------|----------|---------------|
| **v1** | 기본 베이스라인 | ~0.45 | XGBoost 기본 적용, 모든 원천 변수 사용 |
| **v2** | 클래스 가중치 | ~0.52 | 불균형 데이터 대응, sample_weight 적용 |
| **v3** | Top50 피처 선택 | ~0.58 | 변수 선택으로 과적합 방지 |
| **v3.5** | Hybrid Top150 + 도메인 FE | ~0.62 | 도메인 지식 기반 파생변수 6개 추가 |
| **v4** | 희귀 세그먼트 특화 + 2단계 분류 | **0.688** | 계층적 분류 전략, 고급 피처 15개 |

## 🛠️ 활용 방법

### 1. **신규 팀원 온보딩**
- 순서대로 읽으면 프로젝트 전체를 이해 가능
- 각 의사결정의 배경과 근거 학습

### 2. **버전 업그레이드 참고**
- v5 개발 시 v4 → v5 발전 과정 참고
- 실패한 시도를 반복하지 않도록

### 3. **유사 프로젝트 적용**
- 다른 고객 세그먼트 분류 문제에 적용
- 피처 엔지니어링 방법론 재사용

### 4. **논문/발표 자료 작성**
- 체계적인 실험 과정 문서화
- 결과 재현을 위한 근거 자료
