### 1️⃣ 마케팅 전략 문서 작성
**파일명**: `01_MARKETING_STRATEGY.md`

#### 📌 필수 포함 내용

**A. 프로젝트 맥락 (Context)**
- v1 → v2 → v3 → v3.5 → v4 전체 발전 히스토리
- 각 버전별 주요 개선사항 및 성능 변화
- v4 최종 성과: Macro F1 0.688, 희귀 클래스 탐지율 43%
- 2단계 계층적 분류 전략의 차별성

**B. 세그먼트별 분석 (Analysis)**

각 세그먼트(0~4)에 대해 다음 항목 작성:

| Segment | 분석 항목 |
|---------|----------|
| **0 (VIP, 0.04%)** | - 고객 특징 (한도, 사용액, 행동패턴)<br>- 타겟 마케팅 전략<br>- 추천 혜택 및 채널<br>- 기대 효과 및 ROI |
| **1 (High-Value, 0.006%)** | - 성장 잠재력 분석<br>- Segment 0 전환 전략<br>- 육성 프로그램 설계 |
| **2 (중상위, 5.3%)** | - 안정적 수익원 유지 전략<br>- 교차판매 기회<br>- 충성도 프로그램 |
| **3 (중위, 14.6%)** | - 메인카드 전환 전략<br>- 사용 빈도 증대 방안<br>- 업그레이드 인센티브 |
| **4 (일반, 80.1%)** | - 대량 고객 효율적 관리<br>- 휴면 재활성화<br>- 자동화 마케팅 |

**C. 예산 및 ROI 시뮬레이션**
- 세그먼트별 투자 예산 배분
- 기대 수익 계산
- 분기별 실행 로드맵

**D. 실행 전략**
- 필요 기술 인프라 (모델 API, CRM 시스템 등)
- 성과 측정 KPI
- 리스크 및 대응 방안

### 2️⃣ PPT 슬라이드 구성안 작성
**파일명**: `02_PRESENTATION_OUTLINE.md`

#### 📌 권장 슬라이드 구조 (20~25장)

```
[Opening]
1. 표지 - 프로젝트 제목, 발표자, 날짜
2. 목차 - 전체 구성 개요

[Part 1: 프로젝트 배경]
3. 비즈니스 문제 정의 - 왜 세그먼트 분류가 필요한가?
4. 데이터 개요 - 400,000명, 8개 카테고리, 165개 피처
5. 핵심 과제 - 극심한 클래스 불균형 (0.046% vs 80%)

[Part 2: 기술적 여정]
6. v1~v4 발전 타임라인 (인포그래픽)
7. v4 혁신: 2단계 계층적 분류 아키텍처
8. 성능 비교 차트 (v1: 0.45 → v4: 0.688)

[Part 3: 시각화 증거] ⭐ 핵심
9. 세그먼트별 고객 분포 (파이 차트)
10. 주요 피처 중요도 Top 20 (막대 그래프)
11. 세그먼트별 평균 사용액/한도 비교 (막대 그래프)
12. 희귀 클래스 탐지율 비교 (v3.5 vs v4)
13. Confusion Matrix (v4 최종 성능)
14. Stage1 Precision-Recall 곡선

[Part 4: 마케팅 전략]
15. 세그먼트별 고객 페르소나 (5개 슬라이드 또는 1개 통합)
16. 타겟 마케팅 전략 매트릭스
17. 예산 배분 및 ROI 시뮬레이션
18. 분기별 실행 로드맵

[Closing]
19. 기대 효과 - 연간 순수익 증대, 고객 만족도 향상
20. Q&A

[Appendix]
21. 기술 상세 (모델 구조, 하이퍼파라미터)
22. 참고 문헌 및 관련 문서
```

#### 📊 슬라이드별 시각화 요구사항

- 데이터 기반 설득력 강화
- 복잡한 숫자를 직관적으로 전달
